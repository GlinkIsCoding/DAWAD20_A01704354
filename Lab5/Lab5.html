Preguntas a responder:

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Se recomienda validar mediante JavaScript los datos insertados por el usuario. Así, si el usuario cometió un error al rellenarlo, se le notifica de forma instantánea, sin esperar a hacer movimientos en el servidor.[1]

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Quitando a la sección de validación de input el atributo onsubmit, o sentenciar a la función que llama el onsubmit con return true.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque notificar los errores con JavaScript de manera inmediata, mejora la satisfacción del usuario y mejora su experiencia en general cuando no es un usuario malintencionado el que visita el sitio web. Además siempre es bueno proteger lo que pueda entrar a tu servidor por medio de los inputs de formularios, es una buena práctica de seguridad que no hay que descuidar.
