Internet, HTTP y sus métodos

Hay que saber diferenciar que una cosa es el Internet y otra la World Wide Web. El primero es una enorme red de dispositivos y computadoras conectadas entre sí; la segunda es una colección de páginas que se almacena en esa red.

Un "Uniform Resource Locator" o URL, es una serie de caracteres que responden a un formato estándar para localizar recursos en Internet. Son identificadores únicos.

Los MIME Types o "Multipurpose Internet Mail Extensions" son la manera estándar de mandar contenido a través de la red. Especifican tipos de datos como texto, audio, etc. que los archivos contienen. Permiten incluir archivos de imágenes y sonido en un documento HTML.

El protocolo más representativo es HTTP, el cual usa distintos métodos, como los enlistados a continuación:

  • GET: Pedir un recurso en un servidor web (conteniendo metadatos y el mismo recurso)
  • HEAD: Pedir los encabezados o metadatos de un recurso y no el recurso como tal.
  • POST: Enviar información y recursos al servidor para que se almacene en un elemento del mismo. Se usa comúnmente en los formularios.
  • PUT: Crea un nuevo recurso en el servidor web; o en caso de que ya exista, lo reemplaza.
  • PATCH: Aplicar modificaciones parciales a un recurso.
  • DELETE: Solicitar al servidor que borre un determinado elemento.

El método POST debe usarse al enviar un formulario HTML porque un request de POST se suele enviar por un formulario HTML y resulta en cambios en el servidor, es el método adecuado para el tipo de información, pues se envían ocultos en la petición.

El método GET es aquel que se utiliza cuando a través de un navegador Web se accede a una página con un URL.


Errores y recomendaciones

Algunas preguntas frecuentes que te encontrarás al trabajar con HTML:

Un servidor web me devolvió una respuesta HTTP con código 200. ¿Qué significa esto? ¿Hice algo mal?
Significa que la petición fue recibida correctamente y puede ser entendida y aceptada por el servidor. No ocurrió ningún error, es un código de respuesta OK.		 ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Puede que sea responsabilidad del desarrollador o del usuario. El error 404 indica que se trata de un enlace roto, defectuoso o que ya no existe. Pudo haber sido que el usuario ingresara una dirección incorrecta, que el desarrollador lo haya configurado así a propósito por seguridad o que efectivamente haya cometido un error.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Es muy probable que sí. Es un error interno en el servidor del sitio web. Puede tratarse de algún error de permisos incorrecto en un script PHP, o un deficiente manejo de errores en la secuencia de comandos.		 ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)?
Significa que todavía se provee soporte para el atributo, pero pronto ya no lo tendrá, por lo que no es recomendable usarlo.

Algunos elementos que ahora tienen mejor manejo en HTML5 con CSS, y que están desaprobados son:

  • applet: Definía un applet
  • center: Definía texto centrado
  • frame: Definía un marco
  • noframes: Definía una sección sin marcos

HTML y los Sistemas de Información

Las diferencias principales entre HTML 4 y HTML 5 son que HTML 4 no hace distinciones de estilo, pues no se apega a un diseño de ingeniería de software por capas. Además, se percibe HTML5 como un conjunto de tecnologías que incluye JavaScript, CSS y HTML en sí.

Los principales componentes de estructura y estilo de una tabla son los siguientes:

  • table: define una tabla
  • th: la cabecera de la tabla
  • tr: define una fila en la tabla
  • td: define celdas en la tabla
  • caption: define el nombre o título de la tabla
  • colgroup: esencial para un grupo de columnas con un formato específico
  • col: especifica características de las columnas de un colgroup element
  • tfoot: define el pie de la tabla

Aunque en versiones anteriores se podían aplicar varios atributos a la etiqueta table, hoy solo se permite el atributo border, con valores de 0 o 1.

Los principales elementos y etiquetas de una forma HTML5 son los siguientes:

  1. form: etiqueta que define un formulario.
  2. method: control sobre cómo se envían los datos recogidos, mediante programación con PHP, ASP, JavaScript, etc.
  3. input type = text: los inputs son campos que debe rellenar el usuario. Definen el tipo de campo y aquí se refiere a crear un campo tipo de texto.
  4. input type= submit: campo de envío y el diseño suele ser un botón.

Ciclo de vida de los sistemas de información:

  1. Planificación: se definen requerimientos y funcionalidades, en conjunto entre desarrolladores y contacto con el cliente.
  2. Definición de requerimientos: Documentar los requisitos específicos del software a desarrollar. Se hacen diversos bocetos que se prueban antes de empezar a programar.
  3. Diseño: Selección de la arquitectura más adecuada para el desarrollo del proyecto.
  4. Implementación: El desarrollo propio del producto después de las pruebas aplicadas a clientes con los bocetos del programa.
  5. Prueba: desempeño del programa para verificar existencia de fallas o posibilidad de mejora.
  6. Despliegue: lanzamiento del producto al mercado.
  7. Mantenimiento: eventualmente se le puede realizar mantenimiento al producto.

Ciclo de desarrollo de los sistemas de información:

  1. Planificación de proyectos: permite saber qué se debe hacer y quién lo hará. Evidencia obstáculos relevantes del proyecto.
  2. Control de Proyectos: permite preparar y seguir planes de acción.
  3. Estudio de Factibilidad: determina si se puede ofrecer solución automatizada al problema con beneficios tangibles e intangibles. Toma en cuenta costos de la construcción.
  4. Análisis: define qué va a hacer el nuevo sistema. Identifica usuarios y se lleva a cabo análisis de requisitos.
  5. Diseño: define cómo solucionará el sistema los requerimientos planteados.
  6. Implementación: Programación, traducir el diseño en instrucciones.
  7. Actualización: corrige o mejora sistemas existenes.
