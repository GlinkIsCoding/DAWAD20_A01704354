Preguntas

¿Qué importancia tiene AJAX en el desarrollo de RIA's (Rich Internet Applications?

Porque su concepto es cargar y renderizar una página, luego mantenerse en esa página mientras scripts y rutinas van al servidor buscando, en background, los datos que son usados para actualizar la página solo re-renderizando la página y mostrando u ocultando porciones de la misma.

Las aplicaciones de escritorio tienen una riqueza y respuesta que parecía fuera del alcance en Internet.

El motor de AJAX permite que la interacción del usuario con la aplicación suceda asincrónicamente (independientemente de la comunicación con el servidor). Así el usuario nunca estará mirando una ventana en blanco del navegador y un icono de reloj de arena esperando a que el servidor haga algo.

¿Qué implicaciones de seguridad tiene AJAX? ¿Dónde se deben hacer las validaciones de seguridad, del lado del cliente o del lado del servidor?

La interactividad incrementada en una aplicación web significa un incremento de XML, texto y tráfico general de HTML. Esto lleva a exponer aplicaciones de back-end que no habían sido vulnerables anteriormente o, si hay insuficiente protección del lado del servidor, darle a usuario no autenticados la posibilidad de manipular sus configuraciones de privilegios. Todo esto causa un mayor riesgo en que hackers puedan obtener acceso a muchas URLs escondidas que son necesarias para que las peticiones de AJAX sean procesadas.

Es recomendable que las validaciones de seguridad se realicen e implementen tanto del lado del cliente como del lado del servidor.

¿Qué es JSON?

JSON, cuyo nombre corresponde a las siglas JavaScript Object Notation o Notación de Objetos de JavaScript, es un formato ligero de intercambio de datos, que resulta sencillo de leer y escribir para los programadores y simple de interpretar y generar para las máquinas. Es un formato de texto completamente independiente del lenguaje, pero utiliza convenciones que son ampliamente conocidos por los programadores, entre ellos: C, C++, C#, Java, JavaScript, Perl y Python. Dichas propiedades hacen de JSON un formato de intercambio de datos ideal para usar con API REST o AJAX.

Podemos concluit que es un formato común para 'serializar' y 'deserializar' objetos en la mayoría de los idiomas.
