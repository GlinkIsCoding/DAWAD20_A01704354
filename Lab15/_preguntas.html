Preguntas

¿Por qué es una buena práctica separar el modelo del controlador?

Porque se enfoca en separar las responsabilidades. También permite reutilizar código editando la parte del modelo, controlador o vista que requiere un cambio, sin necesidad de actualizar cada lugar donde es utilizado. Finalmente, porque se reduce y elimina incluso el uso de código de servidor y presentación en el mismo lugar (evitar código Espagueti).

¿Qué es SQL injection y cómo se puede prevenir?

Es una vulnerabilidad que permite al atacante enviar o "inyectar" instrucciones SQL de forma maliciosa y malintencionada dentro del código SQL programado para manipular la base de datos, poniendo en peligro a todos los datos almacenados. La finalidad es poder modificar el comportamiento de nuestras consultas a través de parámetros no deseados, falsificando identidades y obtener y divulgar información relevante.

Se pueden realizar pruebas de SQL Injection y automatizar todo el proceso para detectar y aprovechar las vulnerabilidades. Una herramienta para esto es SQLMap.

Adicionalmente, se pueden supervisar las modificaciones automáticas en las aplicaciones comprobando y filtrando los métodos y parámetros que utilizan las aplicaciones para acceder a la base de datos. Los datos transferidos siempre deben estar en el formato de datos esperado.

También se debe proporcionar una protección completa al servidor. Esto se logra instalado y activando solo aquellas aplicaciones y servivios que son realmente relevantes para la base de datos, eliminando todas las cuentas de usuario que no sean necesarias e instalando todas las actualizaciones relevantes para el sistema y el programa.
