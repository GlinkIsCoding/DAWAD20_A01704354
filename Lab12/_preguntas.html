Preguntas

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Porque session_unset() solamente borra las variables de una sesión, la sesión sigue existiendo. session_destroy() nos permite borrar toda la sesión en cuanto el usuario decide salir; es importante para liberar los recursos del servidor y por cuestión de implementar una mejor seguridad.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Las sesiones se mantienen por un periodo de tiempo determinado, como una hora, en el servidor, o hasta que se cierre el navegador. Las cookies siempre que sean aceptadas por el navegador se mantienen en el cliente (navegador del usuario), hasta que sean eliminadas manualmente por el usuario.

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Se usan funciones de fecha y tiempo para concatenárselo al nombre del archivo, y se genera un nombre único dependiendo del momento de subida.

¿Qué es CSRF y cómo puede prevenirse?

CSRF o Cross Site Request Forgery es una manera en que un usuario con malas intenciones explota la autenticación de un usuario registrado en un sistema a través de código incrustado en una página que visitan.

Se puede prevenir generando un valor aleatorio cuando el usuario visita el sitio y guardándolo como cookie. Cada formulario y post request deberá incluir este valor aleatorio para que el servidor compare la cookie real con la generada en el formulario.
